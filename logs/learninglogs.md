# Learning Logs

## Iteration 1: Structure

In the first iteration, the focus was on building the basic structure of the to-do list site using semantic HTML. The page was organised with a main content area, a form for adding tasks, a filter section, a stats area, and a list to display tasks. This stage was mainly about creating a clear layout that would be easy to style and easy to connect to JavaScript later. I learned how important it is to start with a strong HTML foundation because it makes later development much easier.

## Iteration 2: Functionality

In the second iteration, the site became interactive through JavaScript. I added the ability to create tasks, mark them as complete, delete them, filter them by status, and reset the list without refreshing the page. I also added editing so tasks could be updated after being created, and a task details popup so users could store extra information like date, time, location, and notes. High-priority tasks were also sorted to the top, followed by medium and low priority tasks. During this stage, I learned more about DOM manipulation, event handling, and how to keep the interface updated when the user interacts with the page.

## Iteration 3: Polishing

In the third iteration, the focus shifted to design and user experience. The background was made more vibrant, and the site was redesigned with a cleaner, more modern glassmorphism-inspired layout. The app was placed inside a centered card panel, spacing and alignment were improved, and the buttons, inputs, task items, and modal popup were styled to look more polished and professional. Finally, the CSS was moved out of the HTML file and into a separate stylesheet inside a `css` folder, which made the project more organised and easier to maintain. This stage showed me how much visual polish can improve the feel of a project, even when the core functionality stays the same.
